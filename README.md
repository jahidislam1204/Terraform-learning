# Terraform Learning

Practice Terraform project that provisions a simple AWS environment and demonstrates core infrastructure-as-code concepts.

## Overview

This repository creates a basic AWS infrastructure including network components and a single EC2 instance with Nginx installed via user_data.

## Features

- Custom VPC
- Public and private subnets
- Internet Gateway
- NAT Gateway
- Route tables
- Security Groups
- EC2 instance (Nginx auto-installed using user_data)

## Requirements

- Terraform 1.0+ (recommended)
- AWS account and credentials configured (AWS CLI or environment variables)
- IAM permissions to create VPCs, subnets, gateways, NAT, EC2, elastic IPs, and security groups

## Quick Start

1. Configure AWS credentials:
    - export AWS_ACCESS_KEY_ID=...
    - export AWS_SECRET_ACCESS_KEY=...
    - export AWS_DEFAULT_REGION=us-east-1

2. Initialize Terraform:
    - terraform init

3. Review plan:
    - terraform plan -out=tfplan

4. Apply:
    - terraform apply "tfplan"
    - or: terraform apply -auto-approve

5. Destroy resources when done:
    - terraform destroy -auto-approve

## Variables

Typical variables to configure (in terraform.tfvars or via CLI):
- region
- vpc_cidr
- public_subnet_cidrs
- private_subnet_cidrs
- instance_type
- key_name (for SSH access)
- ami_id (Ubuntu/AMI ID for region)

## Outputs

Common outputs provided:
- public_instance_ip
- public_instance_dns
- vpc_id
- public_subnet_ids

Use `terraform output` to view values.

## SSH Access

If a key pair is configured:
- ssh -i /path/to/key.pem ubuntu@$(terraform output -raw public_instance_ip)

## Notes

- State is local by default. Configure a remote backend for team use.
- Adjust security group rules to match your security requirements.
- Nginx is installed via user_data; check EC2 system logs for provisioning details.
